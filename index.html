2 Análisis FODA

Fortalezas, Oportunidades, Debilidades y Amenazas del proyecto de app web para la cafetería.

Fortalezas
Costo de desarrollo y operación prácticamente nulo
Firebase ofrece base de datos en la nube gratuita y en tiempo real
Interfaz accesible desde cualquier dispositivo (responsive)
Permite digitalizar pedidos y reducir errores manuales
El menú se actualiza en tiempo real sin recargar la página
Panel admin intuitivo, sin necesidad de conocimientos técnicos
Oportunidades
Escalar a pagos en línea con Stripe o MercadoPago
Agregar sistema de reseñas y calificaciones de productos
Implementar programa de fidelización (puntos por compra)
Notificaciones push cuando el pedido está listo
Integración con redes sociales para mayor visibilidad
Expandir a otras sucursales usando el mismo sistema
Debilidades
Requiere conexión a internet para funcionar (no offline)
Sin integración de pagos en línea en la versión inicial
El administrador debe actualizar el menú manualmente
No incluye sistema de inventario ni control de stock
Diseño básico sin animaciones avanzadas ni PWA
Amenazas
Competencia de apps consolidadas (Rappi, PedidosYa, etc.)
Posibles cambios en los precios de Firebase
Cuotas gratuitas de Firestore pueden agotarse con alto tráfico
Resistencia al cambio por parte del personal de la cafetería
Cortes de internet afectan la operación del sistema
3 Evaluación de Riesgos

Riesgos identificados con su probabilidad, impacto y estrategia de mitigación.

Sin acceso a internet
Prob: Media Impacto: Alto
Contratar un plan de datos de respaldo o instalar una conexión secundaria en la cafetería
Error en configuración de Firebase
Prob: Media Impacto: Alto
Documentar paso a paso la configuración y mantener backup del archivo de configuración
Cuota Firestore agotada
Prob: Media Impacto: Medio
Monitorear uso en Firebase Console y optimizar consultas; migrar a plan Blaze si el volumen crece
Resistencia del personal al sistema
Prob: Media Impacto: Medio
Capacitación breve (30 min) y guía rápida impresa para el panel de administración
Cambio de API de Firebase
Prob: Muy Baja Impacto: Medio
Fijar versión del SDK de Firebase en el proyecto y revisar changelogs periódicamente
Dominio o hosting caído
Prob: Baja Impacto: Medio
Firebase Hosting garantiza alta disponibilidad (SLA 99.9%); alternativa en Netlify como backup
